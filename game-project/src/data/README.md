# data/

Dados de design em JSON/TS, separados da lógica de código.
Objetivo: balancear o jogo (custos de craft, atributos de recursos, etc.)
sem precisar mexer nos systems.

Exemplos a criar quando o tema estiver definido:
- `resources.json` — lista de recursos coletáveis e seus atributos
- `recipes.json` — receitas de crafting (input -> output)
- `buildables.json` — o que pode ser construído e seu custo
